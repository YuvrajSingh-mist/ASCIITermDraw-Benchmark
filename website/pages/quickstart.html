Quick Start

Setup, Rendering, Generation, and Judging

This page mirrors the benchmark workflow in the repository: create the environment, render PNG references, run a smoke test, run full generation, and then run the DeepEval judge.

Setup Back to Overview

Environment Setup

Install Python and Node dependencies, then create a local credential file. Scripts auto-load .env.local and .env.

uv sync
npm install
npx playwright install chromium
cp .env.example .env.local
FIREWORKS_API_KEY=...
FIREWORKS_ACCOUNT_ID=...

Render Reference PNGs

Render every .ascii artifact into a diagram-only PNG so image-conditioned edit tasks have consistent source images.

uv run python -m scripts.rendered.render_all

Smoke Test

Run a synchronous shared-module spot check before spending time on full-benchmark batch inference.

uv run smoke \
  --model accounts/fireworks/models/qwen3p7-plus \
  --tasks tasks \
  --outputs outputs/smoke-qwen3p7-plus \
  --sample-count 5 \
  --seed 7 \
  --reasoning-effort none \
  --network-retries 5

Full Generation

Run the full benchmark via Fireworks' synchronous chat-completions transport (one request per task). Category 3 tasks automatically attach source images.

uv run run-model \
  --model accounts/fireworks/models/your-vision-model \
  --tasks tasks \
  --outputs outputs/your-run \
  --reasoning-effort none \
  --network-retries 5

Scoring

Judging goes through DeepEval BaseMetric against OpenAI or Anthropic, with structured JSON output where the main score is geval_structural_score + geval_semantics_score. This is the only scoring path — there is no separate deterministic/text-heuristic step.

uv run judge-geval \
  --provider openai \
  --model your-judge-model \
  --tasks tasks \
  --outputs outputs/your-run \
  --results results_your_run.csv \
  --num-judgments 10

Website Build

The published site is generated from the repository tree. Rebuild its data and preview locally before publishing.

node website/tools/build_site_data.mjs
.venv/bin/python -m http.server 8000 -d website
