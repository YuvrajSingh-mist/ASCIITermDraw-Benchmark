Quick Start

Setup, Rendering, Generation, and Judging

This page mirrors the benchmark workflow in the repository: create the environment, render PNG references, run a smoke test, launch batch generation, run the assertion cross-check, and then run structured VLM judging.

Setup Back to Overview

Environment Setup

Install Python and Node dependencies, then create a local credential file. Scripts auto-load .env.local and .env.

python3.10 -m venv .venv
.venv/bin/python -m pip install --upgrade pip
.venv/bin/python -m pip install requests pydantic openai instructor
npm install
npx playwright install chromium
cp .env.example .env.local
FIREWORKS_API_KEY=...
FIREWORKS_ACCOUNT_ID=...

Render Reference PNGs

Render every .ascii artifact into a diagram-only PNG so image-conditioned edit tasks have consistent source images.

.venv/bin/python scripts/rendered/render_all.py

Smoke Test

Run a synchronous shared-module spot check before spending time on full-benchmark batch inference.

.venv/bin/python scripts/smoke_generate.py \
  --model accounts/fireworks/models/qwen3p7-plus \
  --tasks tasks \
  --oneshot oneshot \
  --outputs outputs/smoke-qwen3p7-plus \
  --sample-count 5 \
  --seed 7 \
  --reasoning-effort none \
  --network-retries 5

Batch Generation

Run the full benchmark with Fireworks batch inference. Category 3 tasks automatically attach source images.

.venv/bin/python scripts/run_model.py \
  --model accounts/fireworks/models/your-vision-model \
  --tasks tasks \
  --outputs outputs/your-run \
  --oneshot oneshot \
  --reasoning-effort none \
  --network-retries 5

Scoring

Run the assertion-based structural cross-check first, then VLM judging with structured JSON output where the main score is structural_score + semantics_score.

.venv/bin/python scripts/eval.py \
  --outputs outputs/your-run \
  --tasks tasks \
  --results results_your_run.csv
.venv/bin/python scripts/judge/run_vlm_judge.py \
  --model accounts/fireworks/models/your-judge-model \
  --tasks tasks \
  --outputs outputs/your-run \
  --results results_your_run.csv

Website Build

The published site is generated from the repository tree. Rebuild its data and preview locally before publishing.

node website/tools/build_site_data.mjs
.venv/bin/python -m http.server 8000 -d website
